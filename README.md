## 📌 About the project

JavaBank is a functional mini banking system that runs entirely in the terminal. The project was created with an educational purpose: every class and method was designed to teach Java concepts in a practical and natural way, without artificially forcing language features into the codebase.
The current version uses no frameworks or databases, keeping the focus on core Java fundamentals and Object-Oriented Programming. As my knowledge evolves, future versions of the project will include more advanced features, database integration, and external libraries.

If you're learning Java and want to see concepts like Inheritance, Polymorphism, Interfaces, and Abstract Classes applied in a real and cohesive project, this is the perfect project to study and recreate to solidify your understanding.

---

## ✅ Features

- Create accounts (Checking or Savings)
- View all registered accounts
- Deposit and withdraw funds with validation rules
- Transfer money between accounts
- Check balances with account-specific details
- View transaction history and account statements
- Close accounts
- Simulate savings yields and checking account fees
- View overall bank statistics

## 🧠 Concepts covered

### Java Fundamentals
| Concept | Where it's used |
|---|---|
| Primitive types (`int`, `double`, `boolean`) | Attributes of the `Account` and `Client` classes |
| Reference Types (non primitive) (`String`, objects, arrays) | `transactionHistory: String[]`, `accountHolder: Client` |
| Input validation using Scanner | `Main` - All methods `read*()` |
| Conditional statements and the ternary operator | Validation, status, overdrafts alert |
| Switch-case | Main menu implemented in `Main` |
| Loops (`while`, `do-while`, `for`) | Main menu floew, inout validation and array iteration |
| Reference-type arrays | `Acconut[] account` inside the `Bank` class |
| Garbage Collector | `accounts[i] = null` during account closure |
| Custom constructors | Used across all classes |

### OOP (Object-Oriented Programming)
| Concept | Where it's used |
|---|---|
| Interface | `BankOperable` — interface defining banking operations |
| Abstract class | `Account` — shared base class, cannot be instantiated directly |
| Inheritance | `CheckingAccount` and `SavingsAccount` extends `Account` |
| Polimorfism | `calculateInterest()` and `withdraw()` with different behavior depending on the type |
| Override | `CheckingAccount` overrides `withdraw()` to support overdraft functionality |
| `super()` | Constructors in the subclasses call the `Account` superclass constructor |
| `instanceof` + casting | `Bank` Identifies the actual object type for specific operations |
| Composition | `Account` has a `Client` |

## 🗂️ Structure
 
```
javabank/
├── src/
│   ├── Operable.java        # Interface -- contract defining operations
│   ├── Client.java         # Account holder information
│   ├── Account.java           # Abstract class -- base class for all account types
│   ├── CheckingAccount.java   # Subclass with overdraft support
│   ├── SavingsAccount.java   # Subclass with monthly interest earnings
│   ├── Bank.java           # Accounts array manager

├── out/                     # Compiled .class files generated by javac
├── GUIA.md                 # Comprehensive learning guide
└── execute.sh              # Build and run script
```

## 🚀 How to execute
 
### Prerequisites
JDK 11 or superior instaled
 
```bash
# Clone the repository
git clone https://github.com/OsorioDevx/kantan_bank.git
cd javabank
 
# Compile
mkdir out
javac -encoding UTF-8 src/*.java -d out/
 
# Execute
java -Dfile.encoding=UTF-8 -cp out Main
```
 
Or use the prebuilt script (Linux/macOS):
```bash
chmod +x executar.sh
./execute.sh
```

## 💡 Polymorphism -- core concept ot the project
 
The snippet below in `Bank.java` demonstrates the power of polymorphism in just four lines:
 
```java
for (int i = 0; i < totalAccounts; i++) {
    double resultado = accounts[i].calculateInterest();
    // CheckingAccount → return -14.90  (charges a maintenance fee)
    // SavingsAccount → return +10.00  (pays a 0.5% interest)
}
```
 
The loop does not need to know the concrete type of each account, Java resolves this dynamically at runtime
